# 03. Six-Week MVP Pilot

## Pilot Goal

Launch a controlled ROSE-SEIN MVP that proves four things:

1. users can enter safely
2. the association can publish and coordinate
3. protected messaging works for approved participants
4. the operating team can support and moderate the experience

## Indicative Team

| Role | FTE guidance | Focus |

| product and delivery lead | 0.5 | scope, sequencing, acceptance |
| frontend and integration engineer | 1.0 | app implementation |
| backend and Supabase engineer | 0.75 | schema, auth, messaging |
| editorial and operations owner | 0.5 | content, events, support scripts |
| moderator and support lead | 0.25 | workflows, escalation, safety |
| design and UX support | 0.25 | onboarding and core flows |

## Weekly Plan

### Week 1

- finish profile bootstrap
- confirm canonical MVP scope
- assign operational owners

### Week 2

- wire live articles and events
- define publishing and review workflow

### Week 3

- wire live messaging inbox and thread access
- validate participant-scoped policies

### Week 4

- add reporting and moderation foundation
- draft support, escalation, and privacy scripts

### Week 5

- complete settings and notification preference flows
- add privacy-safe pilot analytics

### Week 6

- run acceptance testing
- perform launch rehearsal
- go live with limited pilot cohort

## KPIs

- account to profile completion rate
- first message thread open rate
- article and event engagement
- support response time
- moderation review time
- zero unauthorized message access incidents
