# 08. Prioritized Roadmap

## Priority Order

### P0. Close the auth and profile gap

- profile bootstrap after magic link
- profile kind selection
- display name and pseudonym preference
- default member role provisioning

### P1. Make content and events live

- article queries from Supabase
- event queries from Supabase
- publishing ownership and review workflow

### P2. Replace messaging mocks

- thread list from live data
- participant-scoped thread access
- server-side send action
- staff-provisioned association-first threads

### P3. Add moderation foundations

- report model
- moderation actions
- audit trail
- escalation scripts

### P4. Complete settings and privacy controls

- notification preferences UI
- privacy explanation and support pathways
- session and account clarity

### P5. Extend parcours carefully

- appointments
- notes
- defer document upload until private storage policy and access logging are ready

## Later-Phase Roadmap

- app-native support-care resource library
- broader community spaces
- scoped volunteer features
- advanced sponsor reporting
- document upload and retrieval
- selective realtime or richer notification delivery

## Explicit Deferrals

Do not prioritize these before the roadmap above:

- open community feed
- mentorship automation
- AI guidance
- full offline sync
- broad social engagement mechanics

## Delivery Rule

Every roadmap step must end with:

- current state documented
- owner assigned
- dependency list confirmed
- do-not-ship criteria checked
