# ROSE-SEIN Association Operations Prompt

```text
Create a sequential agent team to design the best association operations model for ROSE-SEIN.

Project context:
- ROSE-SEIN is run by an association and must support content publishing, event coordination, support messaging, moderation, and privacy-sensitive user care.
- The public website remains the institutional layer.
- The app is intended to become the private operational and support layer.

Mission:
Design the human operating model needed for ROSE-SEIN to work in production, including editorial ownership, moderation, support triage, escalation, release communications, and service quality management.

Focus areas:
- content governance
- event operations
- support inbox handling
- moderation queue handling
- privacy and incident escalation
- volunteer boundary setting
- sponsor and partner reporting ownership

Required deliverables:
- RACI matrix
- support and moderation operating model
- content publishing workflow
- escalation matrix
- weekly and monthly governance cadence
- service-level expectations
- handbook outline for moderators and support operators

Decision principles:
- operational readiness before scale
- moderation traceability before anonymity
- dignity and privacy before responsiveness theater
- clear ownership before automation

Output quality bar:
- recommendations must be specific enough to hand to the association team
- every process must have an owner, trigger, and escalation path
```
