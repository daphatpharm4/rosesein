# 08. UI/UX Strategy

## Design Goal

The interface should feel like an editorial sanctuary, not like a clinical portal and not like a noisy social feed.

## Design System Translation

The existing “Digital Sanctuary” language should remain the base:

- tonal layering instead of hard borders
- soft gradients only for primary emphasis
- generous spacing
- rounded shapes
- strong display-to-body contrast
- mobile-first navigation

## UX Rules for This Product

1. reassure before asking for data
2. explain sensitive features before use
3. keep the first interaction path shallow
4. avoid high-density dashboards
5. let users move between reading, support, and action without cognitive overload

## Information Architecture

| Surface | Primary job | Emotional tone |
| --- | --- | --- |
| Accueil | orient and reassure | calm, welcoming |
| Actualités | surface validated and timely content | informed, steady |
| Messages | private exchange with the association and trusted contacts | warm, controlled |
| Parcours | practical self-organization | discreet, personal |
| Association | explain mission, partners, and ways to engage | credible, human |
| Paramètres | privacy, profile, notification control | transparent, plain-language |

## Layout Direction

### Home

- hero greeting
- one support action block
- one upcoming event block
- one validated content block
- one association message block

### Messaging

- inbox first
- simple scope switch between association and private
- clear unread emphasis
- large tap targets

### Account

- magic-link entry
- explicit privacy reassurance
- immediate post-login profile step

### Parcours

- appointments and notes first
- documents deferred until trust model is fully live

## Accessibility and Fatigue-Aware Design

- large controls and spacing
- strong focus states
- clear labels, never placeholder-only
- restrained motion
- easy language
- no hidden critical actions inside dense menus

## MVP UX Priorities

1. account onboarding with profile kind choice
2. messaging inbox and thread view
3. events and actualités discovery
4. settings for privacy and notifications

## Recommendation

Do not expand navigation before the current key paths are live. Depth can grow later; clarity cannot be retrofitted cheaply.
